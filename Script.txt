"Activity is a sealed interface. The sealed keyword, introduced in Java 17, restricts which classes are allowed to implement this interface. Only CardioActivity, StrengthActivity, and FlexibilityActivity are permitted — the compiler enforces this.

This matters because of pattern matching. Let me show that."

🖥 Show: service/WorkoutService.java — scroll to the switch expression on WorkoutType or Activity.

✏ Annotation: Box around the entire switch expression block.

"Here I'm using a modern switch expression — not a switch statement. It's an expression, so it returns a value directly. The arrow syntax replaces the old fall-through colon syntax.

And because Activity is sealed, if I added a new permitted subtype and forgot to handle it here, the compiler would give me an error. The exhaustiveness check is guaranteed at compile time — that's the power of sealed classes combined with pattern matching."

🖥 Show: service/AnalyticsService.java

"AnalyticsService demonstrates concurrency. I create a fixed thread pool with 4 threads using Executors.newFixedThreadPool."

✏ Annotation: Box around the List<Callable<String>> tasks list.

"I build a list of 6 Callable tasks — each one computes a different analytics metric independently. Callable is like Runnable but it returns a value."

✏ Annotation: Arrow pointing to invokeAll().

"I call invokeAll on the executor — this submits all 6 tasks at once and blocks until every one finishes. It returns a list of Future objects."

✏ Annotation: Arrow pointing to future.get().

"Then I call .get() on each Future to retrieve the result. The key point is that these 6 calculations ran in parallel across multiple threads, not one after another."

"I also pass an unmodifiable copy of the workout list into each Callable to avoid shared mutable state — that was a bug I fixed during development."

Section 10 — Java 25 Preview Features (7:30–9:15)
🖥 Show: Terminal. Type and run:

bash
java --enable-preview Java25Demo
Let the output print. Then switch to Java25Demo.java.

✏ Annotation: Box around void main() — no static, no String[] args.

"This is the instance main method — JEP 445 as a preview in Java 21, standardised in JEP 512 in Java 25. Instead of writing public static void main(String[] args), I just write void main(). The JVM creates an instance of the class and calls this method directly. No boilerplate."

"I compiled this with the --enable-preview flag because I'm on JDK 21. On JDK 25 you wouldn't need that flag — it's standard. That's an important distinction the brief asks us to explain."

Scroll down to ValidatedSession inner class.

✏ Annotation: Box around the validation code if (durationMinutes <= 0) — the lines BEFORE this.sessionId = sessionId.

"This is the flexible constructor body — JEP 482 as preview in Java 21, JEP 513 in Java 25. The rule in older Java was that no statements could appear before field assignments or a super/this call in a constructor. That made it impossible to validate parameters before assigning them.

Here I'm checking that durationMinutes is positive before assigning it. In Java 17 this would be a compile error. With JEP 482 it's perfectly legal."

✏ Annotation: Arrow pointing to where the field assignments happen AFTER the validation block.

"The field assignments — this.sessionId and this.durationMinutes — happen after validation. If the check fails, an exception is thrown and the object is never created in an invalid state. Much cleaner."

Point to the terminal output.

"In the output you can see 'Caught expected error: Duration must be positive, got: -5' — that's the flexible constructor throwing before any field was ever assigned."
